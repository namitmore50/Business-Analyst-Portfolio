# 10_Agile_Sprint_Plan

# Agile Sprint Plan

---

## Methodology

This project followed a **Scrum-inspired Agile methodology** using iterative sprint-based delivery.

### Delivery Model

* Sprint Length: **7 Days**
* Total Sprints: **3**
* Delivery Style: Incremental & Iterative
* Review Method: Sprint-end validation and dashboard demonstration

The Agile approach enabled continuous delivery, rapid problem solving, and stakeholder-oriented prioritization.

---

## Sprint Cadence

| Sprint   | Timeline   | Objective                          |
| -------- | ---------- | ---------------------------------- |
| Sprint 1 | Days 1–7   | Data Engineering + SQL Analytics   |
| Sprint 2 | Days 8–14  | Python EDA + Customer Segmentation |
| Sprint 3 | Days 15–21 | Power BI + BA Documentation        |

---

# Sprint 1 Backlog

## Days 1–7 — Data Engineering & SQL

### User Story Sized Tasks

* Set up project folder structure
* Import Olist datasets
* Clean raw datasets
* Handle null values
* Convert date fields
* Create feature engineering columns
* Create `delivery_days`
* Create `is_late`
* Merge datasets into `master_orders`
* Build SQLite database
* Write 20 SQL business queries
* Export SQL outputs
* Generate business insights report

### Sprint Goal

Deliver a production-ready analytics dataset and SQL insights.

---

# Sprint 2 Backlog

## Days 8–14 — Python EDA & Analytics

### User Story Sized Tasks

* Perform Exploratory Data Analysis (EDA)
* Analyze monthly revenue trends
* Conduct SLA performance analysis
* Perform statistical testing
* Build correlation matrix
* Implement RFM segmentation
* Generate customer segments
* Create revenue forecasting model
* Design A/B experimentation framework
* Estimate campaign ROI

### Sprint Goal

Deliver advanced analytics and predictive business insights.

---

# Sprint 3 Backlog

## Days 15–21 — Power BI & BA Documentation

### User Story Sized Tasks

* Create executive dashboard
* Build customer intelligence page
* Build product profitability page
* Build operations & SLA dashboard
* Create stakeholder reporting page
* Implement KPI governance
* Add What-if simulation
* Create stakeholder matrix
* Create BPMN documentation
* Build RTM
* Perform UAT testing
* Finalize BA documentation

### Sprint Goal

Deliver executive-ready BI platform and complete BA deliverables.

---

## Definition of Done (DoD)

A sprint is considered complete only if:

✅ All sprint backlog items completed
✅ Outputs validated against SQL/Python results
✅ Dashboard visuals tested
✅ Documentation completed
✅ Stakeholder review performed
✅ No critical defects remain
